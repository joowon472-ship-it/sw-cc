## 일정 운영 방침

- 전체 일정 관리는 GitHub Issues와 Milestones 기준으로 진행한다.
- WBS의 각 작업 항목은 1개의 Issue로 등록하여 담당자와 진행 상태를 관리한다.
- 각 Issue는 해당 단계에 맞는 Label과 Milestone을 함께 지정한다.
- 작업 완료 시 Issue를 닫아 마일스톤 진행률에 반영한다.

## 마일스톤 구성

- `M1 기획`: 요구사항 정의, WBS 작성, 비용 산정, 기획서 제출
- `M2 설계`: 유스케이스 다이어그램, 클래스 다이어그램, 설계 패턴 검토
- `M3 구현·보고`: 핵심 기능 구현, Cross-check, 최종 보고서 작성 및 제출
