# 요구사항 정의서

## 기능 요구사항 (Functional Requirements)

| ID    | 요구사항 내용                                                                   | 우선순위 |
| ----- | ------------------------------------------------------------------------- | ---- |
| FR-01 | 사용자는 이메일과 비밀번호를 입력하여 회원가입을 할 수 있어야 하며, 이메일 형식 검증과 비밀번호 최소 길이 조건을 만족해야 한다. | 상    |
| FR-02 | 로그인한 사용자는 강의자료를 업로드할 수 있어야 한다.                                            | 상    |
| FR-03 | 로그인한 사용자는 과목별 시험 날짜를 달력 형식의 입력 UI를 통해 입력할 수 있어야 하며, 과거 날짜는 입력할 수 없어야 한다.  | 중    |
| FR-04 | 시스템은 사용자가 입력한 시험 일정과 업로드한 강의자료를 기반으로 개인 맞춤형 학습 계획을 자동 생성할 수 있어야 한다.       | 상    |
| FR-05 | 시스템은 업로드된 강의자료를 분석하여 핵심 내용을 요약해 사용자에게 제공할 수 있어야 한다.                       | 중    |

## 비기능 요구사항 (Non-Functional Requirements)

| ID     | 요구사항 내용                                                             | 우선순위 |
| ------ | ------------------------------------------------------------------- | ---- |
| NFR-01 | 시스템은 사용자가 모든 입력(시험 일정 및 강의자료 업로드)을 완료한 시점부터 10초 이내에 학습 계획을 생성해야 한다. | 중    |
| NFR-02 | 시스템은 사용자의 비밀번호를 암호화하여 저장해야 하며, 업로드된 강의자료는 해당 사용자만 접근할 수 있어야 한다.     | 상    |
