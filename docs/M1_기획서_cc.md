## 1. 프로젝트 개요

### 프로젝트명

AI 기반 개인 맞춤형 학습 계획 생성 시스템

### 배경 및 문제 정의

대학생들은 과제, 시험 준비, 강의자료 복습을 동시에 관리해야 하지만, 해야 할 일이 많을수록 우선순위를 정하지 못하고 즉흥적으로 학습하는 경우가 많다. 또한 기존 일정 관리 도구는 직접 입력과 정리를 지속해야 하므로 사용 부담이 크고, 계획이 한 번 밀리면 다시 조정하기 어렵다는 문제가 있다.

### 목적

본 프로젝트의 목적은 사용자의 시험 일정과 강의자료를 바탕으로 현실적인 학습 계획을 자동 생성하고, 계획이 지연되더라도 다시 조정 가능한 학습 관리 환경을 제공하는 데 있다.

### 예상 사용자

- 대학생
- 시험과 과제 일정을 체계적으로 관리하고 싶은 학습자
- 직접 계획을 세우는 데 어려움을 느끼는 사용자

### 주요 기능 요약

- 이메일과 비밀번호 기반 회원가입
- 로그인 후 강의자료 업로드
- 과목별 시험 날짜 입력
- 시험 일정과 강의자료 기반 학습 계획 자동 생성
- 강의자료 핵심 내용 요약 제공

### 제공 형태

웹 기반 학습 지원 시스템

### 기능 유형 분류

| 기능 유형  | 기능 목록                               | 개수     | 가중치 | 소계        |

| **합계** |                                     | **10** |     | **48 FP** |

### 공수 산정

- 총 기능 점수: `48 FP`
- 팀 생산성 가정: `12 FP/인월`
- 예상 총 공수: `48 / 12 = 4.0 인월`
- 5인 팀 기준 예상 기간: `약 0.8개월`, 즉 약 `3~4주`

간이 FP 산정 결과, 본 프로젝트는 중간 규모의 학부 팀 프로젝트에 해당한다. 핵심 기능 수가 많지 않지만, 강의자료 처리와 맞춤형 계획 생성이 포함되어 있어 입력, 출력, 내부 데이터 관리 요소가 균형 있게 존재한다.

## 6. AI 활용 계획

### 활용 도구

- ChatGPT
- Claude

### 단계별 활용 방안

- **분석 단계**: 사용자 인터뷰 시뮬레이션, 요구사항 초안 생성, 누락 항목 점검
- **설계 단계**: 유스케이스 및 구조 설계 초안 아이디어 도출
- **구현 단계**: 구현 아이디어 정리, 코드 구조 초안 검토, 오류 원인 탐색
- **검토 단계**: 요구사항 누락 여부, 논리적 모순, 보안상 취약점 점검

### AI 활용 원칙

- AI 생성 결과를 그대로 제출하지 않는다.
- 팀원이 결과를 비판적으로 검토하고 수정한 뒤 반영한다.
- 프롬프트, AI 응답 요약, 수정 내용, 최종 반영 결과를 개인 AI 로그에 기록한다.

### Cross-check 운영 방식

- 13주차에 역할 교환 방식으로 상호 검토를 진행한다.
- 분석가는 설계 산출물을 검토하고, 설계자는 요구사항과 구현 정합성을 검토한다.
- QA/보안 담당은 전체 산출물의 오류, 누락, 보안 관점 문제를 최종 점검한다.
